# Crazyy Fit — Personal Trainer PWA

## What this is
Single-file PWA for a personal trainer managing multiple clients.
Coach PIN: 9090. Clients log in via 4-digit PIN.

## Key architecture
- Single `pwa/index.html` — all CSS, JS, HTML in one file (~676KB, ~12,154 lines)
- Dark theme: #0a0a0a background, DM Mono + DM Sans + Bebas Neue fonts
- localStorage for all data — Supabase optional for cloud sync
- const CLIENTS = [] (empty) — all clients via dynamic_clients in localStorage
- AppState object (L1486) — centralized runtime state
- getLS(key, fallback) helper (L1584) — wraps safeJSON(localStorage.getItem(key))
- Netlify deployment: netlify.toml publishes pwa/ directory

## Login screen (L1398)
- "Enter PIN" → startPinLogin()
- "Join — Start Your Program" → openSignup() (signup/lead capture)
- "Coach Login" → selectCoach()

## Features built
- Multi-client PIN login with fire particle transition
- Coach dashboard: mode badges (inperson/remote/blueprint), view-as-client,
  pause/terminate/restore, adherence %, check-in snippets, compare, Supabase sync
- Onboarding 3-step wizard: profile + BF% + activity level + macro calculation,
  program picker (7 templates), exercise customiser
- Macro engine: Mifflin-St Jeor BMR, activity multiplier, goal detection,
  1g/lb protein minimum, 1.25g/lb for bodybuilder/muscle gain goals
- XP + leveling (10 levels, 5 tiers), fire login transition, accordion home
- Tabata timer with Web Audio haptic sounds (work/rest/countdown/complete)
- SVG tab bar: buildTabBarHTML() with stroke-based icons, 4-char uppercase labels
- AI Program Review: gathers client data, calls Anthropic API (claude-sonnet-4-20250514)
  for program analysis, approved recs applied to client plan
- Coach Notes Log: per-client timestamped notes with flag/delete
- Analytics Panel: cross-client stats (L9614)
- Badges/achievements system: renderBadgesScreen + renderBadgesInCard (L9675)
- Movement Library: coach-managed exercise database (L9807)
- Periodization: phase-based training blocks per client (L9938)
- Progression Suggestions: auto-generated weight/rep suggestions (L10063)
- Email system: Resend API for welcome, credentials, broadcast, coach-message emails
- Signup / lead capture: openSignup() (L10625), openLeads() (L10715)
- Schedule drag-and-drop reorder (L10778)
- Schedule day detail view (L10896)
- Feature screens: Progress, Measurements, 1RM, Meal Plan, Check-In, Calendar,
  Comparison, Macro Analyzer, Badges
- Supabase sync with per-client push notifications (L10989+)

## localStorage keys pattern
- `fit_logs_{cid}` — fitness log entries
- `wl_{cid}_{dayId}` — workout logger sets
- `wl_hist_{cid}_{dayId}` — workout history
- `macro_scores_{cid}` — macro analysis scores
- `meas_{cid}`, `prs_{cid}`, `meals_{cid}`, `checkins_{cid}` — feature data
- `dynamic_clients` — dynamically added clients
- `terminated_clients`, `paused_clients`, `archived_clients`
- `notif_log_{cid}` — notification/message thread
- `client_goals_{cid}`, `goal_cur_{cid}_{goalId}` — goals
- `rest_prefs_{cid}` — rest timer preferences
- `video_links_{cid}` — exercise video links
- `periodization_{cid}` — periodization phases
- `prog_suggestions_{cid}` — progression suggestions
- `movement_library` — coach exercise library
- `signups` — signup leads
- `sb_url`, `sb_key` — Supabase credentials

## Anthropic API usage
Direct browser call in runAnalysis() and openAIReview() to
https://api.anthropic.com/v1/messages. Model: claude-sonnet-4-20250514.
